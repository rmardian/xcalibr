General guidelines


This software is currently under-development and is in beta-release. Please excuse the ugly user-interface and relatively-fragile functionalities (i.e. it might easily break for no obvious reasons). To make the most of use of this tool, please follow the guidelines below:

  1. For tabular files (e.g. parts, assembly plan), please save your files in .csv instead of in .xlsx (or any other format).
  2. For sequence files, please use FASTA format (Genbank is not currently supported, and will be supported in future releases). A single fasta file can contain a single sequence, or multiple sequences. If you have multiple fasta files, compress all the files in .zip format without adding them into another folder.
  3. For parts file, these columns are mandatory: ['name', 'overhang', 'sequence'], and column ['color'] is optional if you want to generate visualization/diagram. Please avoid using different column names.
  4. The mapping file is optional but it can be useful to shorten the name of the parts in your assembly plan. When using this, these columns are mandatory: ['id', 'short_name', 'full_name']. All the values should be unique. Please avoid using different column names.
  5. For any assembly-plan file, these columns are mandatory: ['id', 'name', 'part_1', ..., 'part_n', 'vector']. The number of parts is not fixed as long as consistent. Please avoid using different column names for ['id', 'name', 'vector'], while ['part_1', ..., 'part_n'] can be replaced with other names, but it is recommended to only use alphanumeric characters and replace whitespace with either '_' or '-'.
  6. If you still encounter any error, please refresh the page and try again. If it still did not work, feel free to ask me :) (Rizki.Mardian@ed.ac.uk).
